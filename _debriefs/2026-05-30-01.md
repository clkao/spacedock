## Decisions
- **Vendor-copy-and-exec for status** behind a `Runner` seam (not delegate-to-plugin-path, not native-now) — version-pinned, golden-testable, swappable. Native lands behind the same seam (Stage 5), default flipped in Stage 7.
- **Decision A** — split-root worktree contract: worktree stages isolate CODE only; entity body + stage reports commit to the shared state checkout (two repos, two commit streams). Home: Stage 4 vendored skill.
- **Decision B** — `--new <STDIN` atomic create (mint id + write entity in one op) so seeds are never id-less; dissolves the chicken-and-egg.
- **Decision C** — dispatch derivation uses the entity SLUG, not the filename stem (`index`) — fixes folder-form name/branch/dispatch-file collisions.
- **M1 (native-state-dir): defer the production flip to Stage 7** — flipping in Stage 6 while the live `state:` symlink existed produced a SILENT EMPTY TABLE (`state:` re-applied → nested `entityDir`). Flip + symlink removal must happen together.
- **M2: implement the worktree-copy active-read overlay** — for non-split-root worktree workflows (split-root unaffected), to keep native the global default at full parity.
- **Distribution model** — binary is canonical; per-host PLUGINS register stable agents (`spacedock:first-officer`/`:ensign`) as `--agent` entry points; `spacedock init` installs the plugin (NOT copies skill files into `~/.claude/skills`); `spacedock claude`/`codex` front-end the host launch; **skill↔binary compat is a CONTRACT-VERSION gate** (not raw semver), checked at the front door, `init`/`doctor`, and FO-startup step 0. Plugin route stays as agent registration, version-gated. Codex has its own plugin system; branch installs via `#next` ref (Claude confirmed; Codex TBD).
- **Process:** parallelize IDEATION (7 designs at once), sequence IMPLEMENTATIONS (after a same-package symbol collision); **dual-review high-stakes entities** (validation ensign + adversarial staff audit on its own read-only checkout) — this caught the CRLF hole and the flip-nesting hazard the single reproduction missed.

## Issues — Spacedock
Framework frictions surfaced by dogfooding split-root (most already fixed in the native build; not filed upstream — captured here as the implementation record):
- **Dispatch derivation from filename stem** — `claude-team build` derived worker name/branch/dispatch-file from `index` (folder-form stem) → collisions; entity path placed inside the worktree (wrong for split-root). FIXED in the vendored helper (decisions A/C); native `spacedock dispatch` (7w) carries it forward.
- **`--discover` double-counts** the workflow + its symlinked state checkout. FIXED natively (Stage 6 discovery prune).
- **sd-b32 chicken-and-egg** — `--next-id` enumerates all entities and hard-fails on any id-less one, so an id-less seed is un-bootstrappable through the sanctioned path; `--set id=` succeeds where enumeration fails (inconsistent strictness). RESOLVED by decision B (`--new` atomic create).
- **Shared non-branched `.spacedock-state` is a concurrency bottleneck** — parallel workers race on the one git index; a bare `git commit` swept a sibling's staged file (crossed attribution, no data loss). Mitigation: path-scoped `git commit -m msg -- path`; real fix is tool-managed atomic state commits (folded into decision B / packaging).
- **Same-package parallel merges symbol-collide** — two file-disjoint entities both added a `contains` helper to `package status` → build break at merge. Lesson: sequence anything sharing a Go package.
- Native parity bugs caught by review (all fixed): CRLF files silently dropped; Python `float()` vs Go `ParseFloat` score sort; boot ORPHANS `os.path.join` absolute-worktree semantics; the flip-nesting hazard.
- **`claude-team show-stage-def` not on PATH** — dispatches emit a bare `claude-team` fetch command that isn't reliably on PATH (recurring across ensigns; non-blocking). Self-hosting/install gap.

## Ergonomics — unresolved + CLI improvement ideas (rtk-inspired)
*The captain asked to flag these explicitly.*

**Unresolved ergonomic issues:**
1. **`--workflow-dir` is mandatory and unforgiving.** Omitting it (or running bare `spacedock status --boot`) falls back to the cwd / `dirname(__file__)` and errors confusingly ("README.md has no stages block"). No auto-discovery of the enclosing workflow.
2. **Leaky error on the deprecated `.spacedock-state` path** — post-migration, pointing at the state dir errors with `non-numeric sequential id` (no README → id-style defaults to sequential → sd-b32 ids look invalid) instead of a clear "not a workflow root / use the definition dir."
3. **`--new` is built but not discoverable from the CLI** — only reachable now that native is default, and only as `status --new` (no top-level verb); seed creation via the FO still uses `--next-id` + manual write.
4. **No install step** — `/tmp/spacedock` was cleaned mid-session; `claude-team`/`spacedock` aren't on PATH; every session rebuilds.
5. **No skill↔binary version gate yet** — drift between the (plugin/vendored) skill and the binary is undetected (packaging entity addresses it).

**CLI improvement ideas:**
- **Auto-discovery** — `spacedock status` with no `--workflow-dir` should walk up to the nearest commissioned workflow README (like `git` finds `.git`), instead of the cwd fallback. Biggest single ergonomic win.
- **Actionable errors** — every error should hint the fix (rtk/modern-CLI style): "no workflow here — pass `--workflow-dir` or run inside a workflow"; "this is a state dir, point at the definition dir `docs/dev`."
- **rtk-inspired token frugality** — rtk (the user's Rust Token Killer) wraps dev commands for 60–90% token savings; the agent-facing surface should match: a first-class **`--json`** mode (agents parse without table glyphs/padding), **`--fields`** projection (already partial), and a **`--quiet`** hook mode. Note: rtk's proxy mangled `git log` output and summarized `go test` this session — a tool meant to be agent-driven should emit rtk-survivable / structured output, or ship an rtk profile.
- **`spacedock doctor`** — health + compat check (binary contract-version, plugin compat, `python3` presence for the vendored path, PATH) — inspired by roborev's `check-agents` + its daemon/cli version-mismatch detection.
- **Top-level verbs** — `spacedock new <slug>` (wraps `--new`), `spacedock dispatch` (native claude-team), `spacedock claude`/`codex` (front-ends) — more discoverable than flag-laden `status` subcommands.
- **`spacedock completion`** — shell completion (roborev ships one).
- **Incremental/background review hook** — roborev's post-commit incremental review (session-reuse) is a natural fit for the *validation* stage (shift-left, per-commit review on the code branch). Blocked in-sandbox by `~/.roborev` (needs `HOME` relocation); claude-code is the only working review agent here.

## Observations
- **The dual-review pattern earned its cost twice** — the adversarial staff audit (on its own read-only checkout) caught the CRLF parity hole and the flip+symlink nesting hazard that the thorough single-reproduction validations both PASSED over. For a parity core / production flip, run both.
- **The split-root worktree model is the load-bearing amendment** — "code in the worktree, entity state in the shared checkout, two commit streams" is the thing that made everything else consistent; it should be first-class in the native dispatch contract, not a scope-note.
- **Defer-don't-force on coupled changes** — the flip and the symlink removal were coupled; trying to ship the flip alone (Stage 6) was the one genuinely dangerous moment (silent data-less reads). Deferring the flip to ride with the symlink removal was the right call.
- **My two self-inflicted failures** were both exit-code blindness: a piped `go test | tail` guard masked a broken merge (terminalized it; hotfixed), and `/tmp/spacedock` vanished mid-session. Both are "verify with real signals" lessons now baked into the sprint handoff.
- **The launcher now self-hosts its own dev workflow** — `spacedock status --workflow-dir docs/dev` drove the back half of this very session.
